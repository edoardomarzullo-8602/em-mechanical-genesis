01 — The Project

What is Mechanical Genesis?

Mechanical Genesis is a two-page website dedicated to the history of high-end mechanical watchmaking. The site presents six landmark watches — from the Breguet n.2639 to the Omega Speedmaster — as editorial content.

Beyond the editorial layer, the site implements a complete user profiling system: visitors take an interactive quiz called the Mechanical DNA Test, which assigns them one of three archetypes. This profile is then used to personalise every aspect of the experience on return visits.

Technologies Used

HTML5 for semantic structure — CSS3 with Grid and custom properties for layout and theming — Vanilla JavaScript with no external libraries for all logic — Bootstrap 5 for responsive utilities — Google Fonts (Unbounded + Instrument Sans) for typography.

02 — Structure

Site Architecture

The project is composed of two HTML pages, one shared CSS file, and one JavaScript file. Both pages link to the same external assets, ensuring visual and behavioural consistency.

index.html — The Homepage

The main editorial page. It presents the watch collection across two sections (The Origins and The Tools), contains the hero section with the A/B-tested title, the personalised welcome message, the profile box for returning users, recommended watch cards, the newsletter form, and the footer.

vault.html — The Vault

Hosts the Mechanical DNA Test. Five questions, three choices each. At the end, the user receives their archetype (The Technician, The Explorer, or The Purist) with a matching watch and description. The result is saved in a cookie.

03 — Profiling

User Profiling

The quiz assigns each visitor one of three profiles based on their answers. Each profile corresponds to a watch archetype and drives all personalisation on the site.

The Technician — Pilot

Drawn to engineering precision, high-beat chronographs, and titanium cases. Matched to the Zenith El Primero. Recommended cards: Omega Speedmaster and Cartier Santos.

The Explorer — Diver

Values tool-watch reliability and adventure. Matched to the Rolex Submariner. Recommended cards: Submariner and Blancpain Fifty Fathoms.

The Purist — Dress

Seeks timeless elegance and restraint. Matched to the JLC Reverso. Recommended cards: Reverso and Breguet n.2639.

Quiz Logic

Each answer increments a score counter for one of the three types. At the end, getWinner() finds the highest score using a simple for...in loop — a deliberate choice over Array.reduce() to keep the logic readable and proportionate to the task.

function getWinner() {
  let winner = "Pilot";
  for (let type in scores) {
    if (scores[type] > scores[winner]) winner = type;
  }
  return winner;
}
04 — Cookies & GDPR

Cookie Management

The site uses five cookies. All profiling cookies are gated behind explicit GDPR consent — without cookieConsent=true, no data is written. The GDPR (General Data Protection Regulation) requires websites to request explicit user consent before storing profiling data. The banner appears on every page at first visit and disappears once the user accepts.

05 — A/B Testing

A/B Testing

A/B testing is a technique that shows two different versions of the same page to distinct groups of users, to compare which performs better. In this site, the hero title on the homepage is being tested.

Version A — "Mechanical Genesis"

The original title. Communicates heritage and editorial authority. Targets visitors who already speak the language of horology.

Version B — "Discover Your DNA"

Action-oriented alternative. Creates curiosity and invites interaction with the quiz. Designed to increase engagement from first-time visitors.

On the first visit, assignABVersion() uses Math.random() to assign version A or B with 50/50 probability. The result is saved in the abTest cookie so the user always sees the same version — exactly as in a real A/B test, where consistency is essential for reliable data collection.

// Assigned once at first visit, persisted for consistency
const version = Math.random() < 0.5 ? "A" : "B";
06 — Strategy

Customer Journey

The site implements a complete five-phase customer journey, from first contact to long-term retention.

01

ARRIVE

The GDPR banner appears. The welcome message is neutral. An A/B test version is assigned and saved.

02

PROFILE

The user navigates to The Vault and takes the Mechanical DNA Test. The result is saved in the watchProfile cookie.

03

RETURN

The user comes back to the homepage. The profile box appears, recommended cards are highlighted, and the welcome message is personalised.

04

DEEPEN

Progressive profiling activates from the third visit onward. The visitCount cookie drives increasingly specific messaging.

05

RETAIN

The newsletter form collects the user's contact. The confirmation message is personalised based on their stored profile.

07 — Personalisation

Dynamic Content

Three layers of personalisation activate on the homepage for returning users.

Welcome Message

The subtitle in the hero section changes based on profile and visit count. First visit: neutral. Second visit: recognition. Third visit onward: profile-specific message.

Recommended Cards

Two watch cards are highlighted with a gold border and a "★ Recommended for you" tag. The selection is driven by the stored profile — each archetype has its own pair of watches.

Your Profile Box

A persistent section appears between the hero and the watch grid, showing the user's archetype name and a direct link to retake the quiz — closing the loop and keeping the profile up to date.

08 — Conclusions

What This Project Demonstrates

Mechanical Genesis demonstrates that user profiling, GDPR compliance, A/B testing, and personalised content delivery can be implemented entirely in client-side JavaScript — no backend, no frameworks, no external libraries beyond Bootstrap for layout utilities.

Every technical choice was deliberate: the for...in loop over reduce(), the separation of data from logic, the progressive approach to personalisation. The result is a site that behaves differently for every user, built on five cookies and a clear understanding of the customer journey.
